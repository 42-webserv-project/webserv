## Notes on the parser.

### Finite Internal State Machine.
    Finite State: Meaning it has a finite amount of states that the machine/parser can have. Examples could be; INITIALIZE/PARSING/COMPLETE.
    Or perhaps more defined: STARTLINE/HEADER/BODY/COMPLETE/ERROR

    Internal: Meaning the Machine keeps track of its State itself. 

### The States of the machine.
Let's give an example of a HttpRequest from a client:
```
POST /upload/info.txt HTTP/1.1\r\n
Host: localhost:8080\r\n
User-Agent: Mozilla/5.0\r\n
Content-Type: text/plain\r\n
Content-Length: 15\r\n
Connection: keep-alive\r\n
\r\n
Hello, Webserv!
```

1. StartLine
```POST /upload/info.txt HTTP/1.1\r\n```
2. Headers
```
Host: localhost:8080\r\n
User-Agent: Mozilla/5.0\r\n
Content-Type: text/plain\r\n
Content-Length: 15\r\n
Connection: keep-alive\r\n
```
3. BodyByContentLength
Note how the body needs not end with a `carriage line feed`. It simply checks whether
the received data matches the `Content-Length` header. If not, we know we still need more.
```
Hello, Webserv!
```
