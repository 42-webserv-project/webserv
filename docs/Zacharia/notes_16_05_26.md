## AI OUTPUT OF OPENCODE WRITING SOME OF THE PARSEHEADERS AND PARSEBODY
Implemented real ParseHeaders() and ParseBody().
- ParseHeaders() now:
  - Waits for \r\n\r\n, splits header lines, validates Name: Value format
  - Normalizes header names to lowercase in headers_ (merges duplicates with ,)
  - Parses Content-Length (decimal, overflow-checked)
  - Detects Transfer-Encoding: chunked but rejects it for now (so you can add chunked later without changing the detection surface)
  - Transitions to Body when Content-Length > 0, otherwise completes immediately
- ParseBody() now:
  - Accumulates until input_buffer_.size() >= Content-Length
  - Extracts exactly that many bytes into body_, leaves any extra bytes in input_buffer_
  - Marks parse Complete
Other small supporting changes:
- Added Done internal state to stop Feed() cleanly after completion.
- Removed ctor/dtor std::cout spam and an unused c_str() variable.
- Added minimal helpers for trimming/lowercasing and decimal parsing.
- Updated temp.cpp to demonstrate incremental body feeding; PrintParserRequest() now prints the body too.
