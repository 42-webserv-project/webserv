NOTES:
Each request opens up a new connection. The server closes
the connection after each request. You can see this as a
person calling through the phone with only a single 
question and then hanging up. The person then reads the
response and calls again if it does not have all 
information. THIS IS WHY SOMETIMES IMAGES LOAD WHILE YOU
ARE ALREADY ON THE WEBPAGE!!!

---
WE ARE MAKING A STATE MACHINE PARSER.
This means that we do not have the server wait for a full
request from the user before we call the parser. No, as 
soon as a connection is made we initialize an instance
of the parser. With the bytes we have from this connection
we see how far the parser gets. If the request is in-
-complete, we exit the parser and wait for `poll()` to 
send more information.

The engine calls `parser.consume()` everytime `poll()`
indicates the socket has (more) data.

This is very important as our server is not allowed to
"hang" and, if we have a user with a slow connection, we
are unable to handle other requests because we are dealing
with the slow user before we can take anything else. This
is NOT what we want and therefore HAVE to make a Finite
State Machine. AKA: A fucking awesome way to write a 
parser!!! YESSSSSSSSSSSSSSSSSSSSS!!!!!!
